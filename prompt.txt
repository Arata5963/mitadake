今までの全部いれたフル構成にやっぱりします。

承知いたしました。「ミニマム構成」ではなく、**「24種のドキュメント」に、これまでの議論で出た「チケット管理」と「カスタムコマンド（自動化）」**をすべて組み込んだ、**Claude Codeの能力を極限まで引き出す「フル装備構成」**を提示します。
この構成は、準備に少し時間はかかりますが、一度作ってしまえばAIが「迷子」になる確率を劇的に下げ、開発後半の失速を防ぐ最強の布陣です。
決定版：AI駆動開発「フル装備」ディレクトリ構成

この構成のメリット（なぜフル構成にするのか）
1. AIの「迷い」をゼロにする（段階的開示） ソースにある通り、CLAUDE.md を司令塔とし、必要な時に必要なドキュメント（例：API実装時は 03_api_design.md）だけを参照させることで、AIのメモリ（コンテキスト）を圧迫せず、かつ詳細な指示を守らせることができます。
2. ハルシネーション（嘘）の防止 03_library_docs に「Shadcn UIの正しい書き方」などを置いておくことで、AIが古い情報や間違ったコードを書くのを防ぎます。
3. 品質の自動担保 .claude/commands/ を配置することで、人間がチェックし忘れても、コマンド経由でLintやテストを強制的に実行させることができます。

--------------------------------------------------------------------------------
フル構成を一気に作成するプロンプト
ファイル数が非常に多いため、一度にすべて生成させるとAIがパンクします。以下の**「2段階プロンプト」**を使用してください。
フェーズ1：要件定義と構成の準備
まず、プロジェクトの要件を固め、空のファイル構成を作ります。
# 命令
私のプロジェクトにおいて、Claude Codeでの開発効率を最大化する「フル装備のドキュメント構成」を構築します。
以下のステップで進めてください。

## 目標とする構成
- ルート: CLAUDE.md, tickets/
- .claude/commands/: 自動化コマンド (/save, /pr)
- .claude/00_project/: 要件定義 (2種)
- .claude/01_development_docs/: 技術設計 (15種)
- .claude/02_design_system/: デザイン (5種)
- .claude/03_library_docs/: ライブラリ対策 (4種)

## ステップ1：ヒアリング
まず、私が作りたいアプリの詳細（目的、機能、ターゲット、技術スタック）についてヒアリングを行ってください。
私が「OK」と言うまで質問を続けてください。

## ステップ2：ファイル作成（枠組み）
ヒアリング完了後、上記のディレクトリ構造と、空のマークダウンファイル（中身はタイトルのみで可）を一括で作成してください。
フェーズ2：中身の充填（詳細化）
枠組みができたら、中身を埋めていきます。一度にやると質が落ちるため、カテゴリごとに指示します。
A. 技術設計の充填
# 命令
`.claude/01_development_docs/` 配下の15個の技術設計書について、ヒアリングした要件に基づき、具体的な中身を記述してください。
特に「01_architecture_design.md」と「02_database_design.md」は詳細にお願いします。
B. デザインとライブラリの充填
# 命令
`.claude/02_design_system/`（デザイン）と `.claude/03_library_docs/`（ライブラリ対策）の中身を記述してください。
技術スタックは [Shadcn UI / Tailwind CSS / Supabase / Next.js] (※適宜変更) を前提とします。
C. 司令塔とコマンドの整備
# 命令
1. プロジェクトルートの `CLAUDE.md` を作成し、各ドキュメントへのリンク集（リファレンスマップ）と、開発ルール（Gitフロー、Lint必須）を記述してください。
2. `.claude/commands/` に以下のカスタムコマンドを作成してください。
   - `save.md`: Lintチェックを通してコミットするコマンド
   - `pr.md`: ブランチを切ってPRを作成するコマンド
D. チケットの生成
# 命令
設計書が完成しました。これを基に、実装タスクを `tickets/` ディレクトリに連番ファイル（`01_setup.md`...）として分割・出力してください。
この手順で進めれば、ソースに基づいた**「最も手戻りが少なく、AIが自律的に動ける環境」**が完成します。準備は大変ですが、実装フェーズでの爆速開発が約束されます。
