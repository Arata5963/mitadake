変更をコミットしてください。

## 手順

1. `docker compose exec web rubocop -a`でLintを実行・自動修正
2. Lintエラーが残っている場合は手動で修正
3. `git status`で変更ファイルを確認
4. `git diff`で変更内容を確認
5. 適切なコミットメッセージを作成してコミット

## コミットメッセージ形式

```
<type>: <日本語で変更内容>
```

typeの種類:
- `feat:` 新機能
- `fix:` バグ修正
- `refactor:` リファクタリング
- `test:` テスト
- `docs:` ドキュメント
- `chore:` その他

## 注意事項

- Lintエラーがある状態でコミットしない
- mainブランチの場合は警告して中止
- コミットメッセージは日本語OK
